# v0.8.1: idempotent nvmlInit for nvsandboxutils

Changes since `v0.8.0`.

## NVML Stub

### [fake_nvml.c](https://github.com/ssst0n3/fake-nvidia/blob/v0.8.1/fake_nvml.c) ([5427d5a])

Fix:

- Makes `nvmlInit_v2` (and its weak alias `nvmlInit`) idempotent: a second call
  without an intervening `nvmlShutdown` now returns `NVML_SUCCESS` instead of
  `NVML_ERROR_ALREADY_INITIALIZED`. `libnvidia-sandboxutils.so` (shipped with
  `nvidia-container-toolkit` >= 1.19.0) calls `nvmlInit` twice in a row, so the
  previous behavior made it fail with `ERROR_NVML_LIB_CALL`:

  ```
  ERROR: init 243 result=9
  Failed to init nvsandboxutils: ERROR_NVML_LIB_CALL; ignoring
  ```

  The real `libnvidia-ml`'s init is idempotent and returns `NVML_SUCCESS`;
  re-initialization is a no-op because the fake GPU state is static.
